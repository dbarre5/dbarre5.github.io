Implementation of reservoir simulation in OpenFOAM

Spring 2018

Overview

In near-wellbore regions of a reservoir, fluid flow will begin to depart from Darcy’s Law. Reservoir simulation packages handle this through analytical solutions to simplified geometries and rock properties. However, in more complex completion scenarios, analytical solutions are no longer obtainable. In cases such as these, numerical solutions can be used to analyze the problem [1,2]. Commercial reservoir simulation software packages lack the flexibility needed to simulate such processes. Because of this, it would be beneficial to work on a reservoir simulation toolbox in OpenFOAM [3], a set of C++ libraries for the use and development of numerical solvers. Because of OpenFOAM’s flexibility and availability, a reservoir simulation toolbox could serve as a useful research tool for advanced simulations. Several researchers have utilized OpenFOAM for reservoir simulation [4-6]. While other reservoir simulation toolboxes exist in OpenFOAM, these have not implemented coupling of reservoir simulation with CFD or 1-D pipe simulations. The goal of this project is to develop an open-source, simplified reservoir simulator in OpenFOAM that can be easily coupled to other simulations.

Details

The following set of equations below are discretized and solved using OpenFOAM. A custom solver is made and compiled for this task. The result is simulation of 2-phase incompressible flow through reservoir.

`phirhoc_tpartialp/(partialt)=nabla*(rho/muk(nablap-rhognablaz))+q`

`phipartialS_w /(partialt)=nabla*gamma_wk(lambdap_w-gamma_wnablaz)+q_w`

`P_c = P_o-P_w=0`

`k_(rw)=k_(rw,max)((S_w-S_(wc))/(1-S_(or)-S_(wc)-S_(gc)))^A`

`k_(ro)=k_(ro,max)((S_o-S_(or))/(1-S_(or)-S_(wc)-S_(gc)))^B`
